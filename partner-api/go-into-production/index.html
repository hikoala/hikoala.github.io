Go into production!

Now that you know everything about our Partner API, it is time to know what to do to validate your integration and receive your production token.

Step 1: Check if your staging token is valid#

Before doing anything, it is important to check that your staging token you received is valid or not.

To check your token validity, you should do the following:

curl -X GET "https://staging-partner.api.hikoala.co/quotes" -H "Content-Type: application/JSON" -H "accept: application/JSON" -H "Authorization: Bearer <token>"
note

If you receive an API answer other than 401, you are good to go üëç


Step 2: Send us test Quotes#

To check that you have implemented the right logic to obtain prices from us, you should post at least 5 valid Quotes.

You should use fake data inside your payload for every field except:

  • the Airline IATA
  • the Airports IATA

Caution: Please only use dates in the future for your tests, otherwise the API will return you an error message.

If you have questions or doubt about this, please refer to our Quotes page.

note

Once you have 5 successful quotes, you can go to step 3 üëå


Step 3: Send us test Subscriptions#

Now that you have obtained successful Quotes, we should turn them into Subscriptions.

Just as for the quotes, you should use fake data inside your payload for every field.

If you have questions or doubt about this, please refer to our Subscriptions page.

note

Once you have 5 successful subscriptions, you can either:

  • Go to step 4 if you are selling Koala Flex üê®
  • Go to step 5 if you are selling any other Koala product ü§©

Step 4: Send us test claims (only if you sell our Flex product)#

Now that you have created contracts, we will simulate a booking cancellation by the traveler.

Regarding the data for your payload, we advice that you use:

  • Fake data for the beneficiaryName
  • An IBAN generated by Random IBAN
  • The following BIC: BNPAFRPP

If you have questions or doubt about this, please refer to our Contracts page.

note

Once you have 3 successful claims, you can go to step 5 ü§©


Step 5: Change the environment to prod environment#

Once you have completed all the tasks, please ping us so that we can take a look together. If everything is functioning correctly, we will send you a prod token and you will be good to go! üöÄ

Once you have received your production token, please change your server URL from https://staging-partner.api.hikoala.co into: https://partner.api.hikoala.co/.
