Authentication

Koala chose to use a Bearer token in order to authenticate its partners. To use the Partner API, you will need to provide a specific header to all your requests.

If you have access to this documentation, you should have been provided with a "test" token, the URL of our test server and the link to the documentation (OpenAPI v3). If you are not, please contact support+partners@hikoala.co.

Token scope#

As mention earlier, you received a "test" token that only gives you access to the test server. All operations done with the test server are consequenceless.

danger

All operations done with the production server will have consequences. Actual contracts will be created and you will be charged XXX money. Be careful not to mess your tokens and environments.

Security concerns#

Be extra careful with your tokens. It is your responsibility to secure your token. Anyone who has access to your token can behave as you and we have no way to differentiate you from the XXX.

We will bill you all operations done on your behalf.

tip
  • Do not store your token in your versioning system
  • Use a secret manager
  • Do not share it publicly

HTTP header#

In order to prove your identity, each request should have the Authorization header, with the format Bearer <your token>.

> curl -X GET "https://xxx" -H "Authorization: Bearer <your token>"
{}

Unauthenticated request should result on a 401 error.
