API basics

The Partner API is a RESTful API that uses JSON for representation. All requests must be auhtenticated using an access token.

Interface contract#

Our OpenAPI documentation shows the API contract of both input and output payloads for each route.

Pay a close attention to each attribute definition to make sure your input payload matches the expected format. Otherwise, you will receive a 400 (Bad Request) response from the server.

Status codes#

We use conventional HTTP response codes to indicate the success or failure of an API request. In general, 2xx status codes indicate success, 4xx status codes indicate an error that failed given the information provided (e.g. an invalid authentication token) and 5xx status codes indicate an error with our servers.

Error format#

Errors returned by the API are normalized in order for you to easily understand it. The basic format looks like that:

{
"error": {
"code": 401,
"message": "Invalid token",
"reasons": [
"Token expired"
],
"type": "authentication/token-expired"
}
}

We try to always provide you with an error a human can understand to ease development.

In addition, errors come with a unique technical error code (see type). For instance, the type in the previous example informs us that the token used has expired. You can find detailed reasons why it occurred in our dedicated documentation on errors.
