Subscriptions

How Koala works#

Once the quote is done, it is now possible to proceed to a subscription. You will need to provide these informations:

  • The quote reference token
  • The booking number (from your information system)
  • The client information
  • The travalers informations

For example with the 5K7SXD booking number and the reference_token

{
"data": {
"bookingNumber": "5K7SXD",
"quotes": [
{
"reference": "<REFERENCE_TOKEN>",
"travelers": [
{
"ageRange": "ADULT",
"birthdate": "1964-06-27",
"firstName": "John",
"lastName": "Doe"
},
{
"ageRange": "ADULT",
"birthdate": "1964-01-21",
"firstName": "Jane",
"lastName": "Doe"
}
],
}
],
"client": {
"email": "john.doe@gmail.com",
"firstname": "John",
"lastname": "DOE"
}
}
}

Subscription output#

Invalid payload#

A booking number can only be used once per subscription. If you do not provide the subscription API with a valid payload, you will receive an error 400 (Bad Request), stating the ill-formed field. Please refer to the documentation for the expected input payload format.

Subscription already created#

A subscription corresponds to a unique match between the booking number and the reference_token. To avoid duplicates you will receive an error 409 (Conflict) indicating that the subscription has already been created

{
"error": {
"code": 409,
"message": "Conflict",
"reasons": [
"Subscription has already been created"
],
"type": ""
}
}

Subscription successfully created#

If the payload is valid you will receive a response 201 (Created). This is an example payload that you can receive

{
"data": {
"bookingNumber": "5K7SXD",
"price": {
"amount": 59.97,
"currency": "EUR"
}
}
}
