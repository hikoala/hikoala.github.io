Quotes

Now that we have covered the basics and what a bundle is, it is time for you to understand how to display a price on your purchase funnel.

How Koala works#

As mentioned before, you were assigned bundles. Each time you send us a trip, our quote engine will match every products within a bundle with the underlying trip elements, aggregate those individual results in a quote. You will then receive a quote per bundle.

If we take the previous example, if you have bundles bozk and flux, you will receive:

Quote output#

Invalid payload#

If you do not provide the quote API with a valid payload, you will receive an error 400 (Bad Request), stating the ill-formed field. Please refer to the documentation for the expected input payload format.

Valid payload but not fulling some criteria#

Each product, and, by extension, each bundle will come with the set of criteria to ensure we can generate a contract based on the provided trip. For example, in order to prevent fraud, we may not want to sell Koala's products less than 48 hours before departure, or chose not to cover journeys containing more than 3 flights.

In those cases, you will receive a 200 (OK) answer from a the server but an invalid quote, meaning we can not apply a bundle to the given trip.

tip

If your partnership with Koala involves more than one bundle, one could generate a valid quote and another one an invalid quote. You must only display valid quotes in your purchase funnel.

Invalid quotes come with a field reasons that will help you understand why we could not cover the given trip.
