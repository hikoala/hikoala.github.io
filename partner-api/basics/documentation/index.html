OpenAPI

Our API comes with a low level documentation (OpenAPI) and a swagger UI. Even if the low level documention should be enough to integrate with Koala's API, we highly recommend you to continue reading this documentation to understand the vocabulary and core concepts.

Error handling#

Error returned by the API are normalized in order for you to easily understand errors. An exhaustive list of errors can be found at XXX but basic format looks like that:

{
"code": 401,
"reason": "You do not have access to the resource /v1/partners.",
"error": "#12"
}

We try to always provide you with an error a human can understand to easy development. In addition, some errors comes with a unique technical error code you can find detailed reasons why it occurred in our dedicated documentation on errors.

Data Transfer Object (DTO)#

Our OpenAPI shows the API contract of both input and output payloads for each route. Pay a close attention to each attribute definition to make sure your input payload matches the expected format. Otherwise, you will receive a Bad Request response from the server. This error should carry enough information for you to understand why your payload was rejected, e.g.

{
"code": 400,
"reason": "Field client.first_name cannot be empty.",
"error": "#312"
}
